# Rituals Case Study - Deployment Guide

## GitHub Pages Setup (Easy - 5 minutes)

### Step 1: Create a GitHub Repository
1. Go to [github.com/new](https://github.com/new)
2. Create a new repository named `rituals-case-study`
3. Don't initialize with README/gitignore
4. Click "Create repository"

### Step 2: Upload the File
1. Click "uploading an existing file" (or drag & drop)
2. Upload `rituals_case_study.html`
3. Rename it to `index.html` (important for GitHub Pages)
4. Commit with message "Add Rituals case study"

### Step 3: Enable GitHub Pages
1. Go to repository **Settings**
2. Scroll down to **Pages** section
3. Under "Source", select **main** branch
4. Click **Save**

### Step 4: View Live
Your site will be live at:
```
https://divyanshigupta1603.github.io/rituals-case-study
```

Wait 2-3 minutes for GitHub to build and deploy.

## Customize Later
To make changes:
1. Edit `index.html` directly on GitHub
2. Scroll to bottom, click "Commit changes"
3. Refresh your live site (takes ~30 seconds)

## Alternative Deployments

### Netlify (Drag & Drop - 1 minute)
1. Go to [netlify.com](https://netlify.com)
2. Sign in with GitHub
3. Drag `rituals_case_study.html` onto the deploy area
4. Rename file to `index.html` after upload
5. Get instant free URL

### Vercel
1. Go to [vercel.com](https://vercel.com)
2. Import from GitHub repo
3. Auto-deploys on every push
